# 品牌标识识别系统 - 运行原理文档

## 一、系统概述

本系统用于识别图片中的南方电网品牌标识，并判定其规范类型与布局形式。整体采用**多模型协同 + 规则决策树**的架构：

- **OCR 文本定位**（PaddleX PP-StructureV3）→ 提取文字及坐标
- **Logo 目标检测**（YOLO26-nano + ONNX）→ 定位 Logo 图形
- **ROI 后处理**（几何聚合算法）→ 裁剪品牌标识区域、判定布局
- **规则分类器**（决策树 + JSON 规则库）→ 判定品牌标识规范类型

## 五、各模块运行原理

### 5.1 OCR 定位识别（[ocr_fun.py](file:///c:/Users/12908/Desktop/vi_project/logo_run_func/ocr_fun.py) + [layout_parsing_batch_img_API.py](file:///c:/Users/12908/Desktop/vi_project/logo_run_func/layout_parsing_batch_img_API.py)）

**调用链**：`run_ocr()` → `layout_parsing_batch_img_API.process_images()` → `PaddleX.create_pipeline("layout_parsing")`

**关键配置**（关闭非必要模型以加速）：
- `use_doc_orientation_classify=False`（不识别文档方向）
- `use_doc_unwarping=False`（不矫正扭曲）
- `use_textline_orientation=False`（不识别文本行方向）
- `use_seal_recognition=False` / `use_table_recognition=False`
- `layout_merge_bboxes_mode="small"` + `layout_nms=True`（小框合并 + NMS 去重）
- `engine_config={"enable_new_ir": False, "run_mode": "paddle"}`（禁用 PIR，解决 PaddlePaddle 3.x + oneDNN 兼容性问题）

**输出产物**（保存至 `ocr/<图片名>/`）：
- `<name>_layout_det_res.png` — 版面检测可视化
- `<name>_overall_ocr_res.png` — OCR 可视化
- `<name>_res.json` — 结构化 JSON（含 `rec_texts` / `rec_polys`）
- `<name>.xlsx` / `.html` — 表格结果

**元组提取与合并**（`extract_text_tuples` + `merge_text_tuples`）：
1. 从 `_res.json` 的 `overall_ocr_res` 中读取 `rec_texts` 与 `rec_polys`
2. **朝向推算**：多边形高 > 宽 × 1.5 → 90°（竖排），否则 0°（横排）
3. **同行/同列合并**：
   - 按 orientation 分组
   - 同组内按重叠率（>0.9）+ 边缘对齐（差异 < 平均尺寸 × 0.08）聚类成行/列
   - 行内按间距（< 框尺寸 × 0.03）贪心合并，文本用空格连接，多边形取外接矩形

**输出元组格式**：`(text, poly, orientation)`
- `text`：字符串
- `poly`：四点多边形 `[[x1,y1], [x2,y2], [x3,y3], [x4,y4]]`
- `orientation`：`0`（横排）或 `90`（竖排）

### 5.2 OCR 元组过滤（[tuple_filter.py](file:///c:/Users/12908/Desktop/vi_project/logo_run_func/OCR_Tuple_Information_Filtering/tuple_filter.py)）

**数据源**：[OCR_Filter_Whitelist.json](file:///c:/Users/12908/Desktop/vi_project/logo_run_func/Brand_Resource_Libraries/OCR_Filter_Whitelist.json)

**三级过滤规则**：
1. **空文本过滤** — 文本为空或仅空白 → 丢弃
2. **最小长度过滤** — `len(text) < min_text_length`（默认 2）→ 丢弃
3. **白名单关键字匹配** — 文本须包含至少一个白名单关键字才保留

**白名单分类**（4 大类）：
- `brand_core_names`：品牌核心名称（中国南方电网、CSG、CHINA SOUTHERN 等）
- `subsidiary_keywords`：子公司关键字（公司、局、中心、供电、储能 等）
- `brand_slogans`：品牌口号（万家灯火、南网情深、LIGHT UP 等）
- `non_wholly_owned`：非全资子公司标识

**匹配策略**（兼容 OCR 空格丢失）：
- 中文关键字原文匹配
- 英文关键字大写匹配 + 去空格匹配（如 `CHINASOUTHERN` 也能命中 `CHINA SOUTHERN`）

---

### 5.3 Logo 目标检测（[yolo26_detect.py](file:///c:/Users/12908/Desktop/vi_project/logo_run_func/Yolov26_Logo_Object_Detection/yolo26_detect.py)）

**模型**：`csg_logo_v5.onnx`（YOLO26-nano，纯 CPU 推理，`CPUExecutionProvider`）
**类别**：`["csg_logo"]`（仅检测南方电网 Logo）

**推理流程**：
1. **Letterbox 预处理**：等比缩放至 640×640，不足部分填充灰色 (114,114,114)，保持长宽比
2. **归一化**：BGR → RGB，`/255.0`，转 NCHW 格式
3. **ONNX 推理**：输出 `(1, 300, 6)` 的检测张量
4. **后处理**：
   - 置信率过滤（`conf > conf_thres`，主流程默认 0.85）
   - 坐标还原：`(x - pad) / ratio` 还原到原图坐标系，并 clip 到图像边界内
5. **输出元组**：`(label, (x1, y1, x2, y2), conf)`

### 5.4 ROI 后处理（[logo_roi_processor.py](file:///c:/Users/12908/Desktop/vi_project/logo_run_func/Logo_ROI_Post_Processing/logo_roi_processor.py)）

该模块是几何处理核心，将 Logo + 邻近文字聚合为一个品牌标识 ROI 区域。

#### Step 1：选择最佳 Logo（`_select_best_logo`）
- 先过滤 `conf < 0.5` 的检测框
- 若仅剩 1 个 → 直接采用
- 若全部低于阈值 → 降级取置信率最高的
- 多候选时按**综合打分**选择：
  ```
  score = 面积得分 × 0.4 + 距离得分 × 0.6
  ```
  - 面积得分 = `该 logo 面积 / 最大候选面积`（越大越好）
  - 距离得分 = `1 - 该 logo 到文字群中心的距离 / 最大距离`（越近越好）

#### Step 2：过滤邻近文字（`_filter_nearby_text`）
- 距离阈值 = `logo 对角线长度 × 3.0`
- 文字框与 logo 框的**边到边距离** ≤ 阈值 → 保留为邻近文字

#### Step 3：计算最大 box（`_compute_max_box`）
- 取 logo + 所有邻近文字的外接矩形
- 加 5% 边距（`margin_ratio=0.05`），并 clip 到图像范围内

#### Step 4：裁剪 ROI 图像
- `roi_image = img_bgr[y1:y2, x1:x2]`

#### Step 5：ROI 内坐标重算
- 所有元素坐标减去 ROI 原点 `(ox, oy) = (max_box[0], max_box[1])`
- 输出 `roi_elements` 与 `roi_logo_bbox`（均为 ROI 局部坐标）

#### Step 6：判定布局类型（`_determine_layout`）

| 条件 | 布局类型 |
|------|----------|
| Logo 在文字左侧（`logo.x2 < 文字.x_min`） | **横式** |
| Logo 在文字上方（`logo.y2 ≤ 文字.y_min`）且水平居中（中心 x 偏差 < box 宽 × 0.15） | **中置式** |
| Logo 在文字上方但不居中 | **竖式** |
| 其他情况：`max_box` 宽 ≥ 高 → 横式；否则 → 竖式 | 横式 / 竖式 |

### 5.5 品牌标识分类（[brand_classifier.py](file:///c:/Users/12908/Desktop/vi_project/logo_run_func/Brand_Logo_Norm_Classification/brand_classifier.py)）

**关键字匹配策略**（`_keyword_in_text`，兼容 OCR 空格丢失）：

## 六、核心数据结构

贯穿全流程的统一元组格式：

| 元组类型 | 格式 | 来源 |
|----------|------|------|
| 文字元组 | `(text, poly, orientation)` | OCR + 合并 |
| Logo 元组 | `(label, (x1,y1,x2,y2), conf)` | YOLO 检测 |
| ROI 元素 | `(text, (x1,y1,x2,y2), orientation)` | ROI 坐标重算后 |

- `poly`：四点多边形（OCR 原始输出）
- `bbox`：`(x1, y1, x2, y2)` 外接矩形（Logo 与 ROI 元素使用）
- `orientation`：`0`（横排）/ `90`（竖排）

- 标签位置自动避开上边界溢出

## 九、关键设计要点

1. **多模型解耦**：OCR、Logo 检测、分类各自独立，通过元组数据结构串联，便于单独迭代
2. **规则可配置**：白名单与分类规则均存放于 JSON，无需改代码即可调整关键字
3. **OCR 鲁棒性**：朝向推算 + 同行/同列合并 + 去空格匹配，三层处理应对 OCR 输出不稳定
4. **ROI 聚合策略**：面积 + 距离加权选最佳 Logo，对角线 × 3 过滤邻近文字，几何方法可解释性强
5. **布局判定可解释**：基于 logo 与文字的相对位置关系，规则清晰透明
6. **CPU 友好**：ONNX Runtime 纯 CPU 推理，无需 GPU 即可部署
